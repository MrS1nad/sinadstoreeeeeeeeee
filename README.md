# BeatStore

Django-сайт для продажи битов: витрина с обложками, бесплатное превью,
оплата перед скачиванием, чёрно-белый дизайн.

## Что внутри

- `beats/models.py` — модели `Beat` (бит) и `Purchase` (покупка)
- `beats/views.py` — витрина, страница бита, покупка, скачивание
- `beats/payments.py` — слой оплаты (пример на Stripe, легко заменить)
- `templates/` — шаблоны, `static/css/style.css` — чёрно-белая тема

## Установка

```bash
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate

pip install -r requirements.txt

cp .env.example .env          # и впиши свои значения, особенно Stripe-ключи

python manage.py migrate
python manage.py createsuperuser
python manage.py runserver
```

Дальше зайди в `/admin/`, создай пользователя (если ещё не создал)
и добавь биты: название, цену, обложку, превью-аудио, полный аудиофайл.

## Как это работает

1. Гость видит список битов на главной, у каждого — обложка и цена.
2. На странице бита можно прослушать превью (не полный трек).
3. Кнопка «Купить» доступна только авторизованным — если не вошёл,
   сайт предложит войти.
4. При нажатии «Купить» создаётся `Purchase` со статусом `pending`,
   и пользователя уводит на страницу оплаты Stripe.
5. После успешной оплаты Stripe возвращает пользователя на
   `/purchase/<id>/success/`, где мы проверяем статус сессии и,
   если оплата подтверждена, помечаем покупку как `paid`.
6. Только тогда становится доступна кнопка «Скачать» — она отдаёт
   файл `full_audio` напрямую из `beats/views.py::download_beat`,
   с проверкой, что именно этот пользователь оплатил именно этот бит.

## Оплата: Stripe vs российские платёжные системы

Я использовал Stripe как пример, потому что у него самая понятная
документация и тестовый режим без реального банка. Если тебе нужна
оплата российскими картами, замени содержимое `beats/payments.py` на
интеграцию с ЮKassa, CloudPayments или Robokassa — структура (создать
сессию оплаты → получить ссылку → после оплаты проверить статус)
у всех этих систем одинаковая, поменяются только конкретные вызовы API.

## Важно перед продакшеном

- Смени `DJANGO_SECRET_KEY` и поставь `DJANGO_DEBUG=False`
- Настрой `DJANGO_ALLOWED_HOSTS` под свой домен
- Для реальных денег обязательно подключи Stripe webhook
  (или аналог у другой системы), а не полагайся только на
  редирект на success-страницу — так надёжнее ловить оплату
- Подключи HTTPS, иначе платёжные системы работать не будут
- Файлы `full_audio` физически лежат в `media/`, но раздаются
  только через `download_beat`, а не напрямую — не забудь не
  открывать `media/audio_full/` через веб-сервер (nginx/Apache)
  в обход Django, иначе проверку оплаты можно будет обойти
